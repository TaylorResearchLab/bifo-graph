# BIFO-PPR Pipeline Audit Findings (2026-04-28)

## Background

During §SM5 stability work the week of April 21-27, a discrepancy was identified between the manuscript-canonical KF-CHD WP_CILIOPATHIES result (rank 43, null_z 48.7, q 0.008 — committed in `results/kf_chd/pathway_scores_standard.csv`) and a fresh student-mode REPRODUCE.md re-run (rank 3, null_z ~18.3, q ~0.014). The KF-NBL canonical result reproduced cleanly within ~3% on null_z. This audit identifies why CHD diverged and documents the fixes required to make the pipeline reproducible end-to-end.

## Key findings

### Finding 1: Pipeline cannot run end-to-end as committed

`generate_export_cypher.py` (the production cypher generator, committed in 5d01d32) emits four query files (queries 2-5). However, `clean_files.sh` and `merge_files.sh` reference a fifth file, `kf_{cohort}_pathway_member_nodes.csv`, that is generated by Query 6.

Query 6 exists in the strict-filter exemplar at `examples/strict_filter/kf_*_export_queries.cypher` but was not ported into `generate_export_cypher.py` when the production cypher was committed. As a result:

- A reviewer cloning the repo and running `bash scripts/run_full_pipeline.sh chd` would hit a missing-file error at the cleaning stage.
- The Apr 25 production cypher commit (5d01d32) explicitly claims "A reviewer cloning the repo can now run scripts/run_kf_*_export.sh against the production cypher and reproduce the manuscript numbers without regenerating cypher manually" — but this claim is incorrect because Q6 is missing.

### Finding 2: Canonical CHD edges file dates from before the seed correction

`results/kf_chd/edges_all_noncc.csv.gz` was committed in `8d93617` on Apr 18 18:59. At that time, `data/cohorts/chd/kf_chd_seed_cuis.txt` contained a 55-gene curated set, not the 1,276-gene MAF<=0.001 production seed list.

The CHD seed_cuis.txt was corrected to the 1,276-gene production list in commit `12bf6e9` on Apr 21 06:33. The canonical pathway scores (b5bb2f8, Apr 21 08:40) were generated against the corrected seed list, but the edges file was not regenerated as part of that commit. NBL's edges file was generated correctly from the start (NBL never had a seed-correction event).

The repo's currently-committed CHD edges file (`509d7359...`) is the Apr 18 file that was generated against the 55-gene seed list. Today's REPRODUCE.md re-run uses this file and produces rank 3 / null_z 18.3 — but this is the result of running BIFO-PPR with a 1,276-gene seed CUI list against an edges file that was constructed from a 55-gene neighborhood. The edges file is missing most of the 1-hop neighbors the actual seed list would have generated.

The canonical pathway_scores_standard.csv (rank 43 / null_z 48.7) was apparently generated against a different CHD edges file (regenerated on the HPC against the corrected seed list) that was never committed back to the repo. That regenerated file is no longer recoverable from git.

### Finding 3: NBL reproduces because NBL's inputs are internally consistent

NBL's seed_cuis.txt and edges_all_noncc.csv.gz were both generated against the same 1,395-gene MAF<=0.001 NBL seed list. There was no NBL seed-correction event analogous to the CHD one. Today's REPRODUCE.md re-run on NBL therefore produces near-canonical results (rank 3, null_z 18.4 vs canonical 18.95).

## Conclusion

The CHD vs NBL asymmetry in reproducibility is fully explained by:

- The production cypher generator missing Query 6 (Finding 1)
- The committed CHD edges file having been generated from the 55-gene curated seeds rather than the 1,276-gene production seeds (Finding 2)
- The CHD edges file used to produce the canonical pathway scores was never committed back to the repo

The pipeline is not currently reproducible end-to-end. Both the cypher generator and the committed edges file need to be regenerated against the correct seed lists, with appropriate provenance pinning, before any further claims about KF-CHD pathway results can be defended.

## Required fixes

### Fix 1: Add Query 6 to generate_export_cypher.py

Port `make_query6()` from the strict-filter exemplar. New file: `audit_2026-04-28/pipeline_fixes/generate_export_cypher.py`. Replaces `pipeline/generate_export_cypher.py`.

### Fix 2: Regenerate edges files for both KF cohorts

Run the full export-clean-merge-conditioning-scoring pipeline from scratch for KF-CHD and KF-NBL using the corrected seed_cuis.txt files. NBL is expected to reproduce canonical within numerical noise. CHD will produce new authoritative numbers; canonical CHD pathway scores in the repo are presumed wrong.

### Fix 3: Pin input md5s in REPRODUCE.md

After Fix 2 produces correct reference outputs, update REPRODUCE.md to include md5 checks at each stage (seed file, edges file, conditioned graph, pathway scores). The md5 mismatch that should have caught Finding 2 was not detected because no md5 verification existed.

## Implementation

This audit directory provides:

- `pipeline_fixes/generate_export_cypher.py` — patched cypher generator with Query 6
- `new_export_scripts/run_cohort_export.sh` — generic per-cohort export runner (replaces the cohort-specific scripts)
- `new_export_scripts/run_kf_{chd,nbl,tall,cbtn,rsbd}_export.sh` — five thin wrappers around `run_cohort_export.sh`
- `new_export_scripts/run_one_cohort.sh` — full pipeline for one cohort (export through scoring)
- `new_export_scripts/run_all_cohorts.sh` — driver for all five cohorts (sequential or parallel)
- `new_export_scripts/pre_rerun_setup.sh` — pre-rerun environment check and md5 snapshot
- `snapshots/` — pre-rerun and post-rerun md5 snapshots (populated by pre_rerun_setup.sh)

4. **Adopt new authoritative outputs**: Move scoring outputs from `_rerun_2026-04-28/{cohort}/kf_{cohort}_results/` to `results/kf_{cohort}/` for the manuscript. Commit with explanatory message.
5. **Update REPRODUCE.md** with corrected expected md5s and pathway score values.
6. **Update manuscript** with corrected CHD pathway result and any figures/tables that referenced the canonical numbers.

The Fix 3 md5-pinning approach should prevent recurrence, but review of the broader pipeline-coupling story is warranted as part of the Paper 1 reproducibility supplement.
